# Xiaohongshu Content Strategy

Use this skill when:
- planning an XHS image-note
- preparing title/body/tag structure
- reviewing creator-side note performance
- shaping comment replies with a stable voice

## Layer split

- MCP / platform adapter handles:
  - auth
  - draft creation
  - draft publish
  - note fetch
  - notification fetch
  - comment reply
- This skill handles:
  - title clarity
  - body pacing
  - tag quality
  - image-note structure
  - post-review conclusions

## Title rules

1. Make the topic legible at a glance.
2. Prefer concrete emotional or situational framing over vague poetry.
3. Do not over-pack hot keywords.

## Body rules

Recommended shape:
1. one scene or emotional setup
2. two to four short paragraphs of observation
3. one light closing interaction

## Review loop

When a note underperforms, check:
1. title clarity
2. first image clarity
3. first screen pacing
4. tag relevance
5. comment activation
